# 🚀 AI 产品经理面试官 (AI Product Interviewer)

一个专为产品经理候选人设计的沉浸式 AI 面试模拟平台。通过集成大语言模型，实现高频业务场景的深度对谈与逻辑评估，解决 PM 备考中“实战感缺失”的核心痛点。

---

## 🎯 产品核心亮点 (Product Highlights)

### 1. 业务逻辑建模与评估
* **多维能力覆盖**：深度拆解产品经理核心素质（如：需求优先级排布、商业化路径设计、高压抗压能力等），并将其转化为 AI 智能体的追问逻辑。
* **专业级反馈**：基于 GPT-5 级驱动，实现对回答逻辑性的实时挑战与引导，模拟真实面试中的“压力测试”。

### 2. 沉浸式交互设计 (UX)
* **专业视觉风格**：采用深色系（Slate/Indigo） UI 设计，营造专业、冷静的面试氛围，减少外部干扰。
* **无缝用户路径**：优化单页应用（SPA）切换逻辑，实现从首页预览到面试房间的 0 延迟平滑过渡。

### 3. 技术敏捷落地 (Agile Execution)
* ** MVP 极速交付**：通过 React + Vite + Dify 的现代技术栈，在短时间内完成从产品定义到全球公网部署的全流程。
* ** CI/CD 自动化**：构建基于 GitHub Actions 的自动化流水线，实现代码提交即发布的持续交付能力。

---

## 🛠️ 技术栈清单 (Technical Stack)

| 模块 | 技术选型 | 核心价值 |
| :--- | :--- | :--- |
| **前端框架** | React 18 + TS | 确保复杂交互逻辑的类型安全与高性能 |
| **样式引擎** | Tailwind CSS | 响应式布局与原子化样式管理，确保多端体验一致 |
| **AI 引擎** | Dify.ai | 灵活的 Agent 配置，支撑业务逻辑的快速迭代 |
| **工程化** | Vite 5 + PostCSS | 极致的构建效率，支撑前端样式的精确翻译 |
| **部署发布** | GitHub Actions | 自动化 CI/CD，降低运维成本，确保产品高频更新 |

## 📈 项目复盘与 Owner 意识 (Post-Mortem)

在本项目开发中，我独立完成了从“需求定义”到“上线运维”的全链路管理，并解决了以下关键挑战：
1.  **工程环境调优**：针对 Vite + Tailwind 在静态部署环境下的兼容性问题，通过自定义 PostCSS 插件配置，成功解决了样式丢失的底层 Bug。
2.  **交互风险管理**：针对第三方 AI 接口可能存在的网络波动（非科学上网环境），设计并实现了人性化的 Loading 状态与引导提示，保障了异常情况下的用户体验稳定性。
3.  **CI/CD 路径优化**：根据非标准文件结构，动态调整了 Vite 构建路径与 GitHub Actions 脚本，确立了稳定可靠的发布流程。
